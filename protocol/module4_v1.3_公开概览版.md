# 模块4：自然语言 → 叙事数据包转译协议 v1.3（公开概览版）

## 文件说明

本文件为镜核叙事数据包协议引擎中，模块4的公开概览版本。  
当前公开内容主要用于展示协议方向、基础架构与核心边界，不包含完整运行文本与全部细则。

## 模块定位

模块4用于将作者提供的自然语言内容，转译为结构化叙事数据包。  
其目标不是生成小说正文，而是完成上游结构整理，为后续创作、分析或流程衔接提供中间层输入。

## 输入

模块4当前面向两类输入：

- 上文规则边界摘要
- 当前章节作者自然语言文本

## 输出

模块4输出两部分内容：

- 叙事数据包
- 更新后的上文规则边界摘要

## 叙事数据包基础结构

叙事数据包采用三层结构：

### L1 世界规则
用于识别当前章节文本中直接成立的异常事实。  
例如：异常现象、能力存在、异常装备、异常个体、空间异常等。

### L2 叙事骨架
用于整理当前章节的结构主干。  
例如：章节类型、角色、角色关系、场景、时间节点、事件链、核心冲突、章节目标、推进点、悬念等。

### L3 叙事血肉
用于保留当前章节中的情感驱动、视觉锚点与作者表达特征。  
例如：情绪/动机→行动、强画面锚点、作者原句、节奏句、口号式表达等。

## 当前公开的核心原则

模块4当前遵循以下基础原则：

- 不新增设定
- 不补全未出现信息
- 不将历史边界摘要视为本章直接证据
- 本章 L1 仅收当前文本直接成立的异常事实
- 输出重点在于结构归类，而非文学解释
- 作者原句与高辨识表达应尽量保留

## 上文规则边界摘要

模块4在完成本章转译后，会基于本章 L1 世界规则，生成更新后的上文规则边界摘要。  
该摘要用于后续章节作为参照层使用，但不直接替代当前章节证据。
